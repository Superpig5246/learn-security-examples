Briefly explain the vulnerability and why the secure version prevents the vulnerability.

The insecure version performs a mongoose querry using unsanatized user input which is vulneratble to a noSQL injection. This would allow a malicious user to get all the passwords for every user. The secure version prevents the vulnerability by ensuring that the input is a string and sanatizing the input.
